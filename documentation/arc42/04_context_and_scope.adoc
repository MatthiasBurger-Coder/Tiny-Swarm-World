== Context & Scope

Tiny Swarm World runs as a local automation application from a Linux or WSL
shell. Operators interact with the Python entry point and, when explicitly
approved, with live local infrastructure.

== External Systems

The following systems are outside the Python process and are treated as live
infrastructure:

* LXD or Incus and managed LXC provider nodes;
* Docker Engine and Docker Swarm inside or outside the provider nodes;
* provider networking, netplan, and host/VM networking;
* socat and WSL port-forwarding helpers;
* Portainer and its API;
* Nexus and its API;
* Jenkins, RabbitMQ, SonarQube, Swagger/NGINX, and other deployed stacks;

The default development quality gate does not call these systems. Tests use
mocks, static inspection, architecture checks, and typed configuration
validation unless a future live-infrastructure workflow explicitly approves
integration execution.

== Supported Boundary

The supported operator entry point is:

[source,bash]
----
tiny-swarm-world
----

This command is provided by the editable or packaged Python installation.
`PYTHONPATH=src python3 -m tiny_swarm_world` remains the source-checkout
fallback for diagnostics before installation.

Removed direct setup helpers and host-side orchestration scripts are classified
in `documentation/system/live-operation-surfaces.adoc`. They are not default
workflow entry points.
