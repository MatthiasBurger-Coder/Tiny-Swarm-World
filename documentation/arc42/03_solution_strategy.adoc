== Solution Strategy

Tiny Swarm World uses a safety-first hexagonal architecture.

The implementation strategy is:

* route operator intent through workflow-level CLI commands instead of
  low-level service dispatch;
* select `lxc_native` through Incus as the supported node provider;
* model platform operations as `init`, `reconcile`, `verify`, `reset`, and
  `destroy` workflows with explicit mutating/destructive semantics;
* keep destructive behavior isolated to reset/destroy command catalogs and
  confirmation contracts;
* separate application composition into Platform, Artifacts, and Deployment
  service bundles while keeping them in one Python automation application;
* move stack lifecycle ownership into the Deployment namespace while keeping
  live deployment behavior behind explicit workflow contracts that require
  consent plus observed registry, Portainer, Swarm, external-input, and
  readiness evidence before success is claimed;
* treat command YAML as a typed command catalog with safety classes,
  workflow allow-lists, typed parameters, effects, and verification metadata;
* keep desired inventory separate from observed runtime state and local
  verification evidence;
* fail closed after every mutating apply step unless verification evidence is
  available and `verified`;
* classify direct live-operation surfaces as guarded, transitional,
  deprecated, legacy, or supported assets instead of treating them as
  supported default entry points.

Quality gates validate the Python code, architecture boundaries, type checks,
and tests without running live infrastructure commands.
