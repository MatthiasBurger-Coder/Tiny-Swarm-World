== Runtime View

Processes and workflows during execution.

=== Application Composition

`src/tiny_swarm_world/__main__.py` remains a thin entry point. It parses
operator arguments, checks live-infrastructure consent for mutating service
runs, builds application services through
`src/tiny_swarm_world/infrastructure/composition.py`, and delegates execution to
the selected application service.

The composition root builds `ApplicationServices` as an aggregate of three
service bundles:

* `platform` for current VM, network, Docker, Swarm, preflight, VM IP, and
  socat automation.
* `artifacts` for future artifact registry and publishing workflows.
* `deployment` for future stack lifecycle and deployment workflows.

At this stage, Artifacts and Deployment are composition boundaries inside the
same Python automation process, not independently deployable microservices.
Their bundles are side-effect free when constructed. Existing CLI service
dispatch remains compatible through transitional `ApplicationServices`
properties until the workflow-level CLI slice routes commands directly through
the responsibility bundles.
