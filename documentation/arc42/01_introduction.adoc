== Introduction & Goals

Tiny Swarm World is a Linux/WSL-only Python automation project for operating a
production-like local Docker Swarm environment on managed provider nodes. The
provider path is `lxc_native` through Incus. Multipass is not a
supported node provider. The project provides workflow-level
commands for platform preparation and verification, plus infrastructure assets
for Portainer, Nexus, Jenkins, Apache Pulsar, SonarQube, and Swagger/NGINX.

The current architecture goals are:

* keep normal `platform init`, `platform reconcile`, and `platform expose`
  non-destructive;
* require explicit live-infrastructure consent before mutating local
  infrastructure;
* keep `platform reset` and `platform destroy` behind exact confirmation
  contracts;
* preserve hexagonal dependency direction between domain, application, and
  infrastructure;
* classify command YAML entries through typed safety metadata and workflow
  allow-lists;
* store observed runtime state and verification evidence only under the
  ignored `.tiny-swarm-world/` local state root;
* require verify-after-apply behavior for mutating workflow steps;
* keep live Incus, LXC container lifecycle, Docker
  Swarm, netplan, socat, compose, and service bootstrap commands out of the
  default quality gate.
