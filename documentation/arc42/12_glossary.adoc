== Glossary

`init`::
Creates or prepares missing managed platform resources. Normal init is
non-destructive and must not select destructive cleanup commands.

`reconcile`::
Converges existing managed resources non-destructively. For the default
`lxc_native` platform workflow this means verifying configured managed nodes
and applying only required node lifecycle changes when live consent and
provider readiness allow mutation.

`verify`::
Inspects or validates state without mutating local infrastructure.

`reset`::
Explicit destructive reinitialization workflow. It requires
`RESET_TINY_SWARM_PLATFORM` and is scoped to the managed Tiny Swarm World
local system only: configured LXC-native nodes and marker-verified
project-owned state. Already absent managed nodes count as successful reset
evidence. Missing or mismatched ownership evidence blocks before mutation. It
must not remove unrelated host or provider resources.

`destroy`::
Explicit destructive teardown workflow. It requires
`DESTROY_TINY_SWARM_PLATFORM` and uses the same managed-resource ownership
checks as reset before deleting configured LXC-native nodes.

Live infrastructure consent::
The combination of `--live` and answering `y` at the short interactive
live-infrastructure confirmation prompt.

Command catalog::
Typed YAML configuration under `infra/config/**/command_*.yaml` that declares
command identity, intent, safety class, workflow allow-list, parameters,
effects, and verification metadata. The current product baseline intentionally
contains no command catalog files after Multipass command removal.

Retained VM command-template naming::
Legacy terminology that still exists in dormant command-builder and repository
interfaces. It is not a supported Multipass provider surface and must not be
used to claim current VM or Multipass runtime behavior.

Desired inventory::
Operator-approved target state. It may live in committed configuration under
`infra/config`.

Observed inventory::
Runtime facts discovered from the local environment. It is stored only under
the ignored `.tiny-swarm-world/` local state root.

Verification evidence::
Typed summaries proving an apply or verify result. Evidence must not contain
raw command payloads, stdout, stderr, secrets, tokens, passwords, or multiline
diagnostic dumps.

Service bundle::
An in-process composition boundary such as `PlatformServices`,
`ArtifactServices`, or `DeploymentServices`. These bundles are not independent
microservices.

LXC-native provider::
The default node-provider family for Tiny Swarm World. It uses managed LXC
through LXD or Incus and fails closed when backend readiness or profile
requirements are not satisfied.

Legacy quarantine::
Documentation state for transitional scripts that may still exist but are not
supported workflow entry points. Quarantined scripts must not run in default
quality gates and should be removed or migrated only through explicit cleanup
slices.
