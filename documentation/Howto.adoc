== HowTo

This page collects operational entry points for the current implementation.
Use it together with the user guide pages under `documentation/user_guide`.

== Safe Development Checks

Run the full project gate from the repository root:

[source,bash]
----
./bin/python docker/quality_gate.py quality
----

If `./bin/python` is not available:

[source,bash]
----
python3 docker/quality_gate.py quality
----

== Run The Current Python Entrypoint

[source,bash]
----
PYTHONPATH=docker python3 docker/tiny_swarm_world.py
----

The current active path lists VM IP information. Provisioning steps are present
in the file but commented. Enable those steps only when you intentionally want
to create VMs, change networking, install Docker, or initialize Swarm.

== Prepare Portainer

Portainer preparation is available through:

[source,bash]
----
cd docker/prepare
./prepare.sh
----

This script can prune Docker objects and deploy the Portainer stack. Review the
target Docker context before running it.

== Bootstrap Nexus

Nexus bootstrap is implemented in Python:

[source,bash]
----
python3 docker/prepare/nexus/setup.py
----

Prefer explicit credentials through environment variables:

[source,bash]
----
TSW_PORTAINER_URL=http://localhost:9000 \
TSW_PORTAINER_USERNAME=admin \
TSW_PORTAINER_PASSWORD='<portainer-password>' \
TSW_NEXUS_URL=http://localhost:8081 \
TSW_NEXUS_ADMIN_USERNAME=admin \
TSW_NEXUS_ADMIN_PASSWORD='<new-nexus-password>' \
python3 docker/prepare/nexus/setup.py
----

The script deploys or updates the Nexus stack through Portainer, waits for
Nexus readiness, rotates the admin password, and enables anonymous access.

== Upload Compose Stacks

Bulk stack upload is handled by:

[source,bash]
----
cd docker/compose
./upload_all_stacks.sh -u admin -p '<portainer-password>'
----

This command can delete and recreate stacks in Portainer.

== Service Ports

Common local ports after stack deployment:

* Portainer: `http://localhost:9000`
* Nexus: `http://localhost:8081`
* Swagger editor/UI: `http://localhost:8082`, `http://localhost:8083`
* SonarQube: `http://localhost:9001`
* RabbitMQ: `http://localhost:15672`

Published ports depend on the active compose files and local forwarding setup.
