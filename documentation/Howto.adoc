== HowTo

This page collects operational entry points for the current implementation.
Use it together with the user guide pages under `documentation/user_guide`.

== Safe Development Checks

Run the full project gate from the repository root:

[source,bash]
----
python3 tools/quality_gate.py quality
----

If a virtual environment is active:

[source,bash]
----
python tools/quality_gate.py quality
----

== Run The Current Python Entrypoint

[source,bash]
----
PYTHONPATH=src python3 -m tiny_swarm_world
----

The current active path lists VM IP information. Provisioning steps are present
in the file but commented. Enable those steps only when you intentionally want
to create VMs, change networking, install Docker, or initialize Swarm.

== Run The Guided Setup

The guided setup path is the Python workflow-level command:

[source,bash]
----
PYTHONPATH=src python3 -m tiny_swarm_world setup run --live
----

When prompted, answer `y` only if changing the local LXD/Incus/LXC, Docker Swarm,
networking, service bootstrap, image, and stack state is intentional. Former
direct Portainer and Nexus preparation scripts have been
retired.

== Stack Deployment

Stack deployment is handled by the workflow-level setup command. Compose files
under `infra/config/compose` are source assets, not direct
operator scripts.

== Service Routes

Preferred local service access after stack deployment uses Traefik host routes:

* Service Access landing page and route overview: `https://service-access.tsw.local`
* Portainer: `https://portainer.tsw.local`
* Nexus: `https://nexus.tsw.local`
* Jenkins: `https://jenkins.tsw.local`
* SonarQube: `https://sonarqube.tsw.local`
* Swagger/NGINX: `https://swagger.tsw.local`
* Infisical: `https://infisical.tsw.local`
* Pulsar Manager: `https://pulsar.tsw.local`
* Pulsar Admin API: `https://pulsar-api.tsw.local/admin/v2/clusters`

Traefik owns local ingress ports `80` and `443`. Direct localhost ports are
diagnostic or compatibility access and depend on the active compose files and
local forwarding setup.
