== Multipass Legacy/Fallback Setup On WSL Or Linux

Multipass is no longer the default node provider for Tiny Swarm World. The
default path is `lxc_native` through LXD or Incus. Keep Multipass installed
only when an operator intentionally wants to use the explicit legacy/fallback
provider:

[source,bash]
----
PYTHONPATH=src python3 -m tiny_swarm_world --node-provider multipass_legacy setup run --live
----

The legacy provider must not be selected automatically after an LXD/Incus
readiness failure.

=== Install Multipass

[source,bash]
----
sudo snap install multipass
----

Verify the installation:

[source,bash]
----
multipass version
multipass info
----

=== Configure The QEMU Driver

[source,bash]
----
multipass get local.driver
----

If the driver is not `qemu`, set it explicitly:

[source,bash]
----
multipass set local.driver=qemu
----

Restart Multipass-managed instances if needed:

[source,bash]
----
multipass stop --all
multipass start --all
----

=== Socket Permissions

If the current user cannot access the Multipass socket, ensure the group exists
and add the user:

[source,bash]
----
getent group multipass || sudo groupadd multipass
sudo usermod -aG multipass "$USER"
----

Then set socket permissions:

[source,bash]
----
sudo chown root:multipass /var/run/multipass_socket || true
sudo chmod 660 /var/run/multipass_socket || true
----

Restart the WSL distribution or log out and back in so group membership is
refreshed.

=== Legacy VM Names

The legacy configuration expects these VM names:

* `swarm-manager`
* `swarm-worker-1`
* `swarm-worker-2`

Manual inspection is useful before a legacy run:

[source,bash]
----
multipass list
multipass info swarm-manager
----

=== Destructive Operations

Some commands in `infra/config/multipass` delete and purge VMs. Treat reset
commands as destructive and run them only when you intentionally want to remove
local legacy VM state.
