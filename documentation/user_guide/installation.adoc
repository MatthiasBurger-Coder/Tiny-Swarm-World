= Installation

Tiny Swarm World is developed and operated from a Linux or WSL shell.
Host-specific Windows commands are legacy notes and are not the supported
default path.

== Prerequisites

Install these tools before running project automation:

* Python 3.12 or a compatible Python 3 version
* Git
* Multipass with the QEMU driver
* Docker CLI access from the shell that will run the scripts
* `socat`, when WSL port forwarding is needed

Optional tools for service preparation:

* `curl`
* `jq`

== Python Environment

From the repository root:

[source,bash]
----
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip
python3 -m pip install -r requirements.txt
python3 -m pip install ruff mypy import-linter
----

Use the active virtual environment or the system `python3` interpreter for the
project quality gate.

== Verify The Development Setup

Run the full quality gate from the repository root:

[source,bash]
----
python3 tools/quality_gate.py quality
----

If a virtual environment is active, its interpreter can be used instead:

[source,bash]
----
.venv/bin/python tools/quality_gate.py quality
----

The quality gate sets `PYTHONPATH=src` automatically.

== Multipass Baseline

Check the Multipass driver:

[source,bash]
----
multipass get local.driver
----

The expected driver for WSL/Linux usage is `qemu`. If needed:

[source,bash]
----
multipass set local.driver=qemu
----

Check access:

[source,bash]
----
multipass info
----

== Repository Runtime Model

Python source code lives under `src/tiny_swarm_world` and uses package imports
such as `tiny_swarm_world.application...`,
`tiny_swarm_world.domain...`, and `tiny_swarm_world.infrastructure...`.
Manual Python commands therefore need `PYTHONPATH=src` unless they are run
through `tools/quality_gate.py`.

Example:

[source,bash]
----
PYTHONPATH=src python3 -m tiny_swarm_world
----

== Live Infrastructure Warning

Commands in `src/tiny_swarm_world/application/services/multipass`, `infra/prepare`,
`infra/compose`, and `infra/swarm` can create or delete virtual machines,
change networking, install Docker, or deploy stacks. Run them only when you
intend to change local infrastructure state.
