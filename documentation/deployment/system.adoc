= Deployment

:sectnums:
:toc:

== Purpose

Tiny Swarm World deploys local infrastructure services into a Docker Swarm environment that is usually backed by Multipass virtual machines. The deployment documentation is intentionally limited to the workflows that exist in this repository today.

This project is developed and operated from WSL or Linux shells. Windows-specific shell commands are not the default execution model.

== Deployment model

The repository separates deployment work into three areas:

* `src/tiny_swarm_world/__main__.py` contains the Python entry point for infrastructure orchestration.
* `infra/prepare` contains one-off preparation scripts for services such as Portainer and Nexus.
* `infra/compose` contains compose stacks and upload helpers for Portainer-managed stack deployment.

The active Python entry point currently registers the dependency container and lists Multipass VM IP addresses through `VmIpList`. Infrastructure-changing steps for VM creation, Docker installation, Swarm initialization, network changes, and service deployment are present in the code but commented. Enable those steps deliberately only when the target environment is ready.

== Development gate before deployment work

Run the development quality gate before changing deployment-related code:

[source,bash]
----
python3 tools/quality_gate.py quality
----

When a virtual environment is active, `python` can be used instead of `python3`.

The quality gate covers linting, import architecture checks, architecture tests, type checks, and the pytest suite. It does not start Multipass VMs, initialize Docker Swarm, or deploy live services.

== Portainer preparation

Portainer preparation is started from the repository root:

[source,bash]
----
cd infra/prepare
./prepare.sh
----

`prepare.sh` currently delegates to the Portainer preparation script. Review `infra/prepare/portainer/prepare.sh` before running it against a live environment because it initializes the Portainer administrator account.

== Nexus bootstrap

Nexus setup is implemented in Python under `infra/prepare/nexus/setup.py`. It creates or starts the Nexus stack through Portainer, reads the initial admin password from the container, changes the admin password, and configures the Docker bearer token realm.

Run it from the repository root:

[source,bash]
----
python3 infra/prepare/nexus/setup.py
----

Important environment variables:

[options="header"]
|===
| Variable | Default | Purpose
| `TSW_PORTAINER_URL` | `http://localhost:9000` | Portainer API base URL
| `TSW_PORTAINER_USERNAME` | `admin` | Portainer login user
| `TSW_PORTAINER_PASSWORD` | `adminadminadmin` | Portainer login password
| `TSW_PORTAINER_ENDPOINT` | `primary` | Portainer endpoint name
| `TSW_NEXUS_STACK_NAME` | `nexus` | Stack name in Portainer
| `TSW_NEXUS_URL` | `http://localhost:8081` | Nexus base URL
| `TSW_NEXUS_ADMIN_USERNAME` | `admin` | Nexus administrator user
| `TSW_NEXUS_ADMIN_PASSWORD` | `adminadminadmin` | New Nexus administrator password
| `TSW_NEXUS_INITIAL_PASSWORD_PATH` | `/nexus-data/admin.password` | Initial password path inside the Nexus container
| `TSW_NEXUS_MAX_ATTEMPTS` | `60` | Startup polling attempts
| `TSW_NEXUS_WAIT_SECONDS` | `5` | Delay between polling attempts
|===

== Stack upload

Compose stacks can be uploaded through Portainer with:

[source,bash]
----
cd infra/compose
./upload_all_stacks.sh -u admin -p '<password>'
----

The script uploads the stack definitions from `infra/compose`. It requires a reachable Portainer instance and valid credentials.

== Verification

Useful checks after deployment:

[source,bash]
----
multipass list
docker node ls
docker stack ls
docker service ls
----

Service-specific web interfaces depend on the published stack ports. Typical local defaults are documented in `documentation/Howto.adoc`.

== Operational caution

Deployment scripts can create or modify virtual machines, Docker configuration, Swarm state, network settings, and service credentials. Review the enabled script sections and environment variables before running them against an existing local environment.
