= Infisical Silent Setup

Tiny Swarm World bootstraps the local self-hosted Infisical instance with the
official `infisical bootstrap` CLI command. The greenpath must not require the
first-start setup screen and must not automate browser clicks or write directly
to the Infisical database.

== Configuration

The local service declaration is in `infra/config/services.yml`:

[source,yaml]
----
services:
  infisical:
    enabled: true
    mode: self_hosted
    bootstrap:
      enabled: true
      organization: "Tiny Swarm World"
      admin_email: "admin@tiny-swarm.local"
      admin_first_name: "Tiny"
      admin_last_name: "Admin"
      password_source: "generated_local_secret"
    endpoint:
      internal_url: "http://infisical:8080"
      external_url: "http://localhost:17080"
----

The preferred operator browser route is `https://infisical.tsw.local` through
Traefik. The direct `http://localhost:17080` endpoint is retained as diagnostic
or compatibility access for bootstrap and recovery workflows.

== Required Environment

`./install.sh` generates missing values unless `--no-generate-secrets` is used.
For strict non-generation runs, provide these values through the ignored local
environment files:

* `TSW_INFISICAL_LOGIN_EMAIL`
* `TSW_INFISICAL_BOOTSTRAP_ADMIN_PASSWORD`
* `TSW_INFISICAL_ENCRYPTION_KEY`
* `TSW_INFISICAL_AUTH_SECRET`
* `TSW_INFISICAL_POSTGRES_PASSWORD`
* `TSW_INFISICAL_REDIS_PASSWORD`

For reproducible fixed-password installs, set `TSW_SECRETS_MODE=fixed` or pass
`./install.sh --secrets-mode fixed` and provide every required key from
`infra/config/secrets/infisical-secrets.yaml` in
`.tiny-swarm-world/local/fixed-secrets.env` or `TSW_FIXED_SECRET_ENV_FILE`.
Missing files, missing keys, empty values, and Infisical write failures abort
the run before deployment can continue.

== Generated Local Files

Infisical secrets are written only to ignored local files:

* `.tiny-swarm-world/local/live-installation.env`
* `.tiny-swarm-world/local/fixed-secrets.env`
* `.tiny-swarm/secrets/bootstrap.local.env`

The Infisical bootstrap file contains local `TSW_INFISICAL_*` values and
non-secret browser URL overrides such as `TSW_DASHBOARD_URL` and
`TSW_INFISICAL_URL`. Compose and the bootstrap service map the Infisical values
to Infisical's `INITIAL_BOOTSTRAP_*` runtime environment names. Do not commit
either file.

== Bootstrap Behavior

The installer runs the official CLI bootstrap behavior equivalent to:

[source,bash]
----
infisical bootstrap \
  --domain=http://localhost:17080 \
  --email=admin@tiny-swarm.local \
  --password=<redacted> \
  --organization="Tiny Swarm World" \
  --ignore-if-bootstrapped
----

The `--ignore-if-bootstrapped` flag keeps reruns idempotent. If the instance is
already initialized, the run records `already_bootstrapped` instead of
recreating users.

== Evidence

Sanitized evidence is written under `.tiny-swarm/evidence/infisical/`:

* `bootstrap-result.json`
* `healthcheck.log`
* `install-summary.md`

Evidence redacts passwords, auth secrets, encryption keys, Redis secrets, and
database connection secrets.

== Reset Behavior

The governed fresh-install reset is still controlled by `./install.sh
--confirm-reset` or the interactive `RESET_TINY_SWARM_PLATFORM` confirmation.
Resetting managed platform state may remove containers and volumes, after which
the next install generates or reuses local ignored secrets and bootstraps
Infisical again.

== Security Notes

Do not print bootstrap passwords, machine identity tokens, auth secrets,
encryption keys, or database connection strings. If the Infisical CLI is
missing, the workflow stops with `infisical_cli_missing`. If Infisical does not
become ready, it stops with `infisical_readiness_timeout`.
